# Step 4 附录：剧本写作规则

本附录供 [agents/script-writer.md](agents/script-writer.md) 使用。单集动笔前必须先做事件链验证，剧本只写可观测的画面与声音。

## 典型错误

> 典型错误1：原文同一段话里同时含多个时间点的信息，改编时直接挪用，导致"信息出现在它的因果前提之前"。原文散文可以，剧本每一帧都是当下，不允许。

> 典型错误2：`△` 画面描写里混入文学比喻（"像灿烂却没底气的星河"）。`△` 段只写摄像机拍得到、麦克风收得到、灯光/美术做得出的视听细节；感受和意象必须转成具体的画面、动作、表情或环境暗示，让观众自己感受，不是被告知。

> 典型错误3：**冗余短场景**。每个`## 场景X`都必须有独立存在的理由——换地点、或有独立戏剧单元（冲突/决策/信息揭露）。纯过渡动作（出门、走路、拿东西）不单独成场，并入相邻场景的△结尾。判断标准：去掉这个场景，下一个场景的信息量是否完整？完整则删，不完整则留。

> 典型错误4：**机械替换文字通讯**。原著里"发了条消息""看了眼微信"等文字传达，默认优先改成电话或视频通话，以增强声音、台词和情绪；但短信、微信、截图、聊天记录、新闻标题、监控文字等如果承担证据、误会、延迟阅读、传播等剧情功能，必须保留为屏幕文字或道具特写，不能为了有声化而破坏信息差和因果链。

> 典型错误5：**台词逻辑断裂**。原文叙述中主语模糊的句子（"说想要见证"——谁想要？），改编为台词时必须选择一个主语，选择后整句逻辑可能不成立。必须重新验证：台词中的每个代词、每个因果连词，在固定主语后是否自洽。

> 典型错误6：**△ 追加推断**。`△` 中每个描述必须是被拍摄对象当下的可观测状态，不含对原因、时长、动机的推断（"——攥了太久""——太紧张了"均属推断）。原因和动机由前后场景让观众自己得出，不由△告知。判断标准：删掉破折号后面的内容，画面信息是否完整？完整则推断多余，不完整则换一个不需要推断就能读懂的细节。

> 典型错误7：**△ 物理词汇偷渡情感**。用物理上可能存在、但叙事上作为隐喻使用的描述（"眼底有一丝光"——物理上是泪液反光，叙事上指"心里亮了一下"；"目光一暗""空气凝固"同理）。判断标准：如果这个描述字面成立（眼底确实有反光），它能否传达编剧想表达的意思？如果不能，说明编剧靠的是隐喻层，不是画面层——应替换为能直接传达该情绪的可观测细节（"眼底有光"→"眼眶泛红"；"目光一暗"→"垂下眼"；"空气凝固"→"没人说话"）。

> 典型错误8：**概括性叙述脱离画面孤立VO化**。原文"每次她都…""八年来总是…""他一直…"这类概括性回忆，需要用蒙太奇视觉化——让观众看见这个模式，而不是被告知。但这不代表这类句子里不能出现VO：蒙太奇里**可以**带着这句概括性心声，穿在某个画面节点里，作为角色在那一帧自然流过的心理。区别在于：VO是蒙太奇内部的一帧（声画咬合），还是蒙太奇外部的总结（声画割裂）。前者是角色的心理活动，后者是编剧在帮观众归纳——只禁止后者。

> 典型错误9：**声画分层不咬合**。△写完、对白写完、VO写完，三者各自独立堆叠，没有在同一时刻咬合——这是散文思维，不是剧本思维。剧本的时间单位是帧，每一帧画面、声音、心声同时发生。正确写法：VO要长在动作里，夹在两个△之间，对应那个动作发生时角色的内心；对白要嵌在画面描写的间隙，不是画面写完再另起一段说话。判断标准：把△、对白、VO三层叠在一起读，应该是同一个瞬间的三个维度，而不是三段先后发生的描述。反例：`△ 他按下拉黑。**明远洲**（vo）：八年了。`（VO在动作后面，像事后注解）正例：`△（特写）拇指悬在屏幕上，停了一秒。**明远洲**（vo）：八年了。△ 拇指按下拉黑。`（VO长在停顿里，是那一秒的内容）

> 典型错误10：**蒙太奇VO压尾**。蒙太奇内部只有△画面，VO全部堆在【蒙太奇结束】之后做总结。这是把蒙太奇当无声幻灯片、把VO当字幕说明的惰性写法——声画割裂，情绪无法叠加。正确做法：每个△画面节点配一句VO，VO补画面传达不了的内心层，声画同时推进，情绪逐帧递进。判断标准：将蒙太奇内的VO单独抽出来读，应是一条从起点到峰值的情绪递进线；若读起来像对画面的总结或重复，则VO位置错了。

## △ 使用判断：写多少才对

△ 不是越少越好，也不是越细越好。判断标准是**这个 △ 对读者/执行者有没有增量信息**。

### 三个原则

**最简原则**：去掉这个 △，意思是否完整？完整则删。台词本身已经能传达的情感和动作，不需要 △ 重复说明。

```
# 不需要写的例子（△ 和台词意思重叠）
△ 小明深情地看着小华。
小明：我爱你。

# 只写台词就够了
小明：我爱你。
```

**不误读原则**：台词/动作与真实意图相反时，必须写 △ 揭示真实态度——否则读者会按字面理解。潜台词场景是 △ 最必要的地方。

```
# 必须写 △ 的例子（表面示好，实则厌恶）
小明：你是我最好的朋友。
△ 他给了小华一个拥抱，但在她身后翻了个白眼。

# 如果不写 △，观众会以为小明是真心的
```

**创意原则**：你想到了一个非常规的动作细节，能表现角色性格、又是导演/演员不一定会自己想到的——这种 △ 值得写，因为它是故事设计的一部分，不写会丢失。

```
# 值得写的例子（创意动作传达信息，不写就没了）
△ 她教一句，呼了他一个巴掌，要他重复。没重复对，再呼一个。
△ 瓜子壳堆满了旁边的桌子。  ← 不写就不知道他等了很久
```

### 判断流程

```
这个 △ 是否去掉意思仍完整？
  → 是：删除（最简原则）
  → 否：是否因为台词/行为与真实意图相反？
       → 是：必须保留（不误读原则）
       → 否：是否是导演/演员想不到的创意设计？
            → 是：保留（创意原则）
            → 否：删除
```

## 改编优先级表

**核心原则**：能演的不说，能闪回的不VO，能蒙太奇的不逐场演。VO是退路不是首选。

| 原文材料类型 | 首选 | 次选 | 退路 | 禁止 |
|------------|------|------|------|------|
| 文学性心理描写 | 闪回（让观众看见） | △ 画面（可观测行为/表情） | VO | — |
| 叙述性旁白 | 蒙太奇 | 闪回 | 旁白VO | — |
| 概括性回忆（"每次…""总是…""八年来…"） | 蒙太奇 | 闪回（单次事件时） | — | VO概括（被告知不是被看见） |
| 角色当前行为由观众未见的往事驱动 | 闪回 | — | VO解释 | — |
| 建立行为模式 | 蒙太奇 | — | — | 多场sequential △（浪费时长） |
| 关键情绪瞬间（爽点/心碎/反转） | 特写+慢镜头 | — | △常规描写 | — |
| 过去信息一句话能说清 | VO | — | — | 闪回（一句话不值得展开） |
| 集末悬念 | 定格 | — | 淡出 | — |
| 已符合视听要求的对白（短/快/信息密度够） | 保留原文 | — | — | — |

**闪回/蒙太奇的两条使用约束：**
- **是反应，不是铺垫**：必须有明确触发点（台词/画面/动作），闪回/蒙太奇紧跟触发点之后。触发点必须在剧本中显式存在。
- **反应式优先于铺垫式**：除非制造戏剧反讽（观众预知，角色不知），否则闪回/蒙太奇放在触发点之后，让观众和角色一起发现过去。

**禁止的改编操作：**
- 角色合并
- 时间线压缩
- 结尾调整
- 删减支线
- 对白因果链断裂（A 的对白包含 B 反应的触发点，触发点必须在对白内自然出现）
- **动作方向改变**：原文"A打B不接"不能改成"B打A"——主动方/被动方的互换会根本性改变人物处境和情绪逻辑

## 视听语言工具箱

**容器/内容原则**：工具箱里的工具分两层——**叙事结构**是容器，**视听元素**是内容。容器内部可以嵌套任何视听元素。蒙太奇里可以嵌对白+VO+特写；闪回里可以嵌完整对话+慢镜头；冷开场里可以嵌任何东西。叙事结构不是特效标签，是组织内容的方式。

**蒙太奇中VO的位置逻辑**：VO必须分散穿插在各个画面节点之间，每句VO对应一个画面，声画同时推进情绪。**禁止**在蒙太奇结束后用一句VO总结整段。正确检查方式：把蒙太奇里的VO全部单独拿出来读，应该是一条情绪递进线，而不是对画面内容的重复或总结。

### 叙事结构（容器）

| 工具 | 格式 | 用途 |
|------|------|------|
| 闪回 | `【闪回】...【闪回结束】` | 记忆/往事的画面化呈现。内部可嵌套对白、VO、特写等——闪回是完整的叙事空间，不是只有画面 |
| 蒙太奇/平行剪辑 | `【蒙太奇】...【蒙太奇结束】` | 同类事件压缩呈现或两条线交替推进。内部可嵌套对白片段、VO、特写——蒙太奇是压缩叙事，不是无声幻灯片 |
| 冷开场 | `【冷开场】...【冷开场结束】` | 集首5-15秒高能片段。内部可嵌套对白、VO、特写、慢镜头等 |
| 梦境/幻觉 | `【梦境】...【梦境结束】` | 主观扭曲的当下感知，区别于闪回。内部可嵌套任何视听元素 |
| 电话/视频分屏 | `【分屏】左：... 右：...【分屏结束】` | 双方画面同时呈现。内部可嵌套对白、特写等 |

### 视听元素（内容）

| 工具 | 格式 | 用途 |
|------|------|------|
| △ 画面描写 | `△ 描述内容` | 场景中观众看到和听到的一切：环境、光线、角色动作、表情神态、行为细节、音效。替代心理描写和文学性叙述 |
| 对白 | `**角色名**：对白内容` | 角色说出口的话 |
| VO | `**角色名**（vo）：对白` | 画外旁白/内心独白，角色在画面但嘴不动 |
| OS | `**角色名**（os）：对白` | 角色在场景空间但不在画面内 |
| 旁白/解说 | `**旁白**（vo）：内容` | 非角色VO，第三人称叙述者 |
| 特写 | `△（特写）描述` | 强调关键细节：表情变化、手部动作、物品特写。短剧高频工具，用于放大情绪和爽点 |
| 慢镜头 | `△（慢镜头）描述` | 放慢关键瞬间，用于爽点强化、情绪定格、关键动作强调 |
| 定格 | `△（定格）` | 画面冻结，集末利器 |
| 画外音效 | `【画外音效】描述` | 观众听到但角色未必听到的声音 |
| 屏幕文字 | `【屏幕文字】内容` | 手机消息、新闻标题、监控文字等 |
| 人物字幕条 | `【屏幕文字】角色名 · 身份标签` | 角色首次入画时屏幕下方弹出姓名+身份标签；**所有有名有姓的角色首次入画均须添加，主角不例外**；同一角色全剧有且只有一次 |
| 转场 | `△（切）`/`△（淡入）`/`△（淡出）`/`△（次日）`等 | 无标注则靠场景标题自然过渡 |
| 时间标记 | `△（X小时前）/△（X分钟前）` | 冷开场结束后的时间回退，标明正叙起点。仅冷开场后使用 |
| 角色首次出场 | `△ 描述内容`（含外貌/身份/此刻行为/状态） | 第一次出场完整描写（角色此刻在做什么、什么状态），后续不再重复 |
| 通讯方式转换 | `**角色名**：对白` / `【分屏】` / `【屏幕文字】内容` | 默认优先电话或视频通话增强声音和情绪；承担证据、误会、延迟阅读、传播功能的短信/微信/截图/文字记录必须保留为屏幕文字或道具 |

## 爽点强化规则

原文有弱爽点时，可在视听语言转换时适度强化（加定格、加周围人反应、加音效暗示）。但不允许凭空制造原文没有的爽点。

## 短剧编剧强制约束

1. **冷开场优先**：每集前 15 秒必须抓住观众，不允许慢热铺垫。优先使用冷开场（规则详见 step3-outline.md），不适合冷开场的集也必须有其他强钩子手段
2. **每集必留钩子**：每集结尾必须有悬念/反转/冲突，且钩子必须可信——观众需要相信"这次真的不同了"才会追下一集，而非"又来了，过两集肯定和好"
3. **信息密度高**：对白要短、快、信息量大，不说废话
4. **情绪外化**：情绪必须通过对白和动作直给，不依赖潜台词
5. **节奏紧凑无闲笔**：每个场景都必须推进剧情或升级冲突

## 短剧语言信息压缩原则

凡需要在极短时间内让观众抓住关系和反转的台词，优先用**"预期+反转"结构压缩在同一句话里**——让观众同时听到"原本应该是什么"和"实际是什么"，落差即钩子。VO、对白、OS均适用。

**具体要求：**
- 用肯定句点出具体关系，不用否定句——`是我未婚妻的闺蜜` 比 `不是她` 信息量大三倍
- 关系词要具体（未婚妻/闺蜜/竹马/亲哥哥），不用泛称（她/他/那个人）
- 语感口语化，像在转述一件荒诞的事，不像在朗读剧情简介

**适用场景：** 冷开场VO、结尾钩子台词、爽点对白、反转揭露、任何需要一句话砸出关系+反转的时刻

**示例：**
- 弱：`结婚当天，站在我身边的不是她。`（否定，模糊）
- 强：`结婚当天，站在我身边的是我未婚妻的闺蜜。`（肯定，具体，关系+反转）
