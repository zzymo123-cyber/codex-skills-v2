# Workflow — 故事转短剧执行流程

本流程只覆盖世情和悬疑两类短剧改编。目标是先把原文结构读准，再把文学叙述转成可执行的视听剧本。

建议流程：类型检测 → 分析 → 大纲 → 类型强度审稿 → 试写第 1 集 + 中后段关键集 → 用户确认 → 批量生成 → script doctor 审稿 → 重写问题集。

## 0. 输入接收

接收用户提供的故事文本、故事名、期望集数或输出路径。

- 无故事名：从标题、核心人物或核心案件中提取。
- 无指定路径：默认输出到当前工作区 `outputs/{故事名}/`。
- 有指定路径：使用用户明确给出的路径。

## 1. 文本完整度判断

调用 [agents/type-detector.md](agents/type-detector.md)，先判断输入是否足以改编。

| 输入状态 | 处理方式 |
|---------|---------|
| 完整故事 | 正常进入类型检测 |
| 短输入（3000-5000 字） | 按材料量减少集数，不硬凑 12 集 |
| 长输入（3-5 万字） | 原则上封顶 20 集，每集可接近 3 分钟 |
| 碎片输入（大纲/梗概） | 可继续，但必须提示情节连贯性需要人工复核 |
| 非小说输入（新闻/案例） | 可继续，按故事事件链处理 |

完整度不合格但用户要求继续时，后续每个质量门都要标注“原文材料不足风险”。

## 2. 类型检测

调用 [agents/type-detector.md](agents/type-detector.md)，输出：

```markdown
主类型：世情/悬疑（置信度 X%）
子类型：家庭伦理/职场博弈/社会底层/都市情感/市井烟火/推理探案/心理悬疑/犯罪惊悚/诡计解谜
副类型：无/世情/悬疑 + 具体补充
导语区域：[原文位置或无]
输入完整度：完整/短输入/长输入/碎片/非小说案例
处理建议：继续/请用户补文本/继续但标注风险
```

经过 [agents/quality-gate.md](agents/quality-gate.md) 的类型判断质量门后，请用户确认。用户确认前不进入分支分析。

## 3. 分支分析

### 世情分支

调用 [agents/worldly-drama-analyst.md](agents/worldly-drama-analyst.md)，必须产出：

- 人物关系压迫图
- 原文事件清单
- 情绪升级曲线
- 主角受压 / 崩溃 / 觉醒 / 反击节拍
- 信息差装置
- 世情戏剧强度表
- 反击合理性表

### 悬疑分支

调用 [agents/suspense-analyst.md](agents/suspense-analyst.md)，必须产出：

- 真相时间线
- 表面时间线
- 线索台账
- 误导层级
- 观众/角色知情矩阵
- 反转节奏表
- 线索便利性风险表

混合类型只做一个主分支，副类型作为补充栏，不画两套完整分析图。

通过分析质量门后，请用户确认关键事件、断点、信息差或线索投放。用户确认前不写分集大纲。

## 4. 分集大纲

调用 [agents/outline-writer.md](agents/outline-writer.md)。

大纲生成原则：

- 集数由原文天然断点、情绪转向、悬念节点和材料量决定。
- 每集必须有戏剧任务。
- 每集必须有结尾钩子。
- 冷开场可选，但前 15 秒必须抓人。
- 禁止按字数机械均分。

一次性内部生成全剧大纲，对外按 3-5 集一段展示。用户修改某段后，从该段起始集开始重刷该段及后续段。

通过大纲质量门后，进入类型强度审稿；强度审稿通过后，再请用户确认全部大纲。

## 5. 类型强度审稿

调用 [agents/quality-gate.md](agents/quality-gate.md) 的“类型强度质量门”，在试写前审稿。

- 悬疑主类型或副类型：执行悬疑强度审稿。
- 世情主类型或副类型：执行世情强度审稿。
- 混合类型：主类型完整审稿，副类型补充检查。
- 强度审稿不通过，不进入试写；返回 [agents/outline-writer.md](agents/outline-writer.md) 或对应分析 agent 修订。

悬疑强度审稿重点：

- 后半段是否每集都有阻力、误判或人物选择代价。
- 关键证据是否过于完整、直接、便利。
- 对峙集是否一次性解释核心真相。
- 线索回收是否有延迟、误读或半真层。
- 终局是否回收人物选择和关系代价，而不是结案说明。

世情强度审稿重点：

- 关系压迫是否具体到利益、面子、控制、亏欠或依赖。
- 受压、崩溃、觉醒、反击是否有触发和铺垫。
- 冲突是否只靠争吵推进。
- 每 3-5 集是否有情绪峰值、关系反转或爽点兑现。
- 终局是否回收尊严和关系代价。

## 6. 试写校准

调用 [agents/script-writer.md](agents/script-writer.md)，优先试写第 1 集，并追加试写 1 集中后段关键集；如果第 1 集承担大量设定铺陈，中后段关键集必须选择最能代表全剧风格的高压/反转集。

试写目标：

- 校准人物对白口吻。
- 校准节奏密度和场景粒度。
- 校准导语、冷开场、VO、OS、闪回、蒙太奇的使用方式。
- 验证事件链、信息差或线索投放没有被破坏。
- 验证类型强度审稿提出的风险已落到具体场面和对白。

试写集必须经过单集剧本质量门。用户确认后再批量生成。

## 7. 批量生成

按已确认大纲逐集生成，每集动笔前先做事件链验证：

```markdown
事件A → 触发条件：[什么信息已成立] → 事件B → 触发条件：[什么信息已成立] → ...
```

每集生成后自查：

- 是否中国影视剧本传统格式。
- `△` 是否只写可观测内容。
- VO/OS/闪回/蒙太奇是否按需使用。
- 有名角色首次入画是否有人物字幕条。
- 信息差、线索、因果链是否按分析表执行。

## 8. Script Doctor 审稿

批量生成后，按类型调用 doctor：

- 悬疑主类型或副类型：调用 [agents/suspense-script-doctor.md](agents/suspense-script-doctor.md)。
- 世情主类型或副类型：调用 [agents/worldly-script-doctor.md](agents/worldly-script-doctor.md)。
- 混合类型：主类型完整 doctor，副类型补充检查。

doctor 必须输出问题集、重写优先级和具体改法。发现问题后，先重写问题集，再进入全剧一致性质量门。

## 9. 质量检查

调用 [agents/quality-gate.md](agents/quality-gate.md) 执行：

- 类型判断质量门
- 分析质量门
- 大纲质量门
- 类型强度质量门
- 单集剧本质量门
- 全剧一致性质量门

不合格时按质量门给出的返回步骤重做，不继续落盘。

## 10. 文件落盘

质量通过后，将文件写入：

```text
outputs/{故事名}/
├── characters.md
├── appearance.md
├── locations.md
├── overview.md
└── script/
    ├── ep01.md
    ├── ep02.md
    └── ...
```

辅助文档按 [assets/characters.md](assets/characters.md)、[assets/appearance.md](assets/appearance.md)、[assets/locations.md](assets/locations.md)、[assets/overview.md](assets/overview.md) 生成。

落盘后复核：

- 文件结构完整。
- 集数与大纲一致。
- 角色、地点、关键道具在辅助文档和剧本中一致。
- 输出路径与用户指定或默认约定一致。
