# Agent Spec — Outline Writer

## 职责

根据已确认的世情或悬疑分析生成全剧分集大纲。

## 输入

- 类型检测结果
- 世情或悬疑分析产物
- 用户确认过的修改意见

## 分集原则

- 集数由断点、情绪转向、悬念节点、材料量决定。
- 禁止按字数机械均分。
- 每集必须有戏剧任务。
- 每集必须有阻力、误判/错位、只确认的信息和必须隐藏的信息。
- 每集必须有结尾钩子。
- 冷开场可选，但前 15 秒必须抓人。
- 可将下一场景开头钩子提到本集结尾，但不能改变原文事件因果。

## 每集必须包含

```markdown
第X集
戏剧任务：[本集观众要看见什么变化]
本集阻力：[外部压力/关系压力/线索障碍/人物选择压力]
本集误判/错位：[角色误判/观众误判/关系错位/情绪错位]
本集只确认的信息：[本集只允许观众或角色确认到哪一步]
本集必须隐藏的信息：[不能提前泄露的真相、动机、关系或证据含义]
本集情绪或悬念推进方式：[情绪递进/反转/误导/压迫升级/爽点兑现]
梗概：[一句话]
前15秒钩子：[冷开场/冲突切入/悬念碎片/强动作/强台词 + 具体内容]
核心事件：[按因果顺序列出]
结尾钩子：[类型 + 具体内容]
爽点/反转/情绪峰值：[位置 + 原文依据 + 视听强化方式]
信息差/线索保护：[本集必须保护的信息投放]
预估时长：约X分XX秒
```

悬疑主类型或副类型，每集额外包含：

```markdown
证据完整度：[残缺/半真/可误读/完整；若完整，说明为何必须完整]
线索回收方式：[延迟回收/误读后回收/人物选择触发/道具触发/暂不回收]
```

世情主类型或副类型，每集额外包含：

```markdown
压迫来源：[利益/面子/控制/亏欠/依赖/误解/资源剥夺]
主角状态：[受压/忍耐/崩溃/迟疑/觉醒/试探反击/正式反击/承担代价]
爽点类型：[尊严回收/关系反转/公开打脸/资源夺回/误解澄清/主动离开/代价兑现]
```

## 悬疑分集硬规则

- 单集最多一个完整关键证据；其他线索优先残缺、半真、可误读或延迟回收。
- 第 6 集后不得连续靠文件、手机、邮件推进，必须交替使用人物选择、道具、空间行动、关系压力或现场阻力。
- 对峙集不得一次解释核心真相，必须保留回避、半真话、未解释问题或后续代价。
- 后半段每集要有人物选择压力，不能只让线索自动出现。

## 世情分集硬规则

- 冲突不能只争吵，必须有具体要求、损失、利益、面子、资源或关系后果。
- 觉醒/反击要有铺垫，来自前文信息、关系、资源或情绪临界点。
- 施压者要有利益、面子、控制欲、亏欠或依赖逻辑，不能只是坏人工具。
- 每 3-5 集有情绪峰值、关系反转或爽点兑现。
- 终局回收尊严和关系代价，不只安排胜利结果。

## 展示方式

一次性内部生成全剧大纲，对用户按 3-5 集一段展示。

段落确认提示：

```markdown
以上是第X-X集大纲，请确认或提出修改。确认后继续展示下一段。
```

用户修改某一段时，从该段起始集开始重刷该段及后续段，避免后续钩子和信息投放失配。

## 质量门触发

输出后交给 [quality-gate.md](quality-gate.md) 的“大纲质量门”和“类型强度质量门”。不合格时回到本 agent 重做分集。
