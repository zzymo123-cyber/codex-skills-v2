# Agent Spec — Quality Gate

## 职责

在类型、分析、大纲、类型强度、单集剧本和全剧一致性阶段做验收。不合格时明确返回哪一步重做。

## 1. 类型判断质量门

通过条件：

- 主类型只能是世情或悬疑。
- 子类型来自 [step1-type-detection.md](../step1-type-detection.md)。
- 副类型有明确叙事作用；没有则写“无”。
- 导语区域没有把正文第一章起点误判为导语。
- 输入完整度判断给出处理建议。

不合格返回：[type-detector.md](type-detector.md)。

## 2. 分析质量门

世情通过条件：

- 人物关系压迫图覆盖主角、主要施压者、关键帮手和关键误解者。
- 原文事件清单按故事发生顺序列出，不先分集。
- 情绪升级曲线标的是情绪转向，不只是事件边界。
- 主角受压 / 崩溃 / 觉醒 / 反击节拍能从原文找到依据。
- 信息差装置写清观众何时知道、角色何时知道和保护要求。
- 世情戏剧强度表写清压迫来源、施压者逻辑和原文依据。
- 反击合理性表写清前文铺垫、可调用资源、触发点和代价。

悬疑通过条件：

- 真相时间线和表面时间线可互相映射。
- 每条关键线索有投放、误读和回收。
- 误导层级不改变原文结局和关键人物行为方向。
- 观众/角色知情矩阵覆盖主要反转。
- 反转节奏表的每个反转都有前置线索。
- 线索便利性风险表写清原文获取方式、改编风险、降便利处理和延迟回收方式。

不合格返回：[worldly-drama-analyst.md](worldly-drama-analyst.md) 或 [suspense-analyst.md](suspense-analyst.md)。

## 3. 大纲质量门

通过条件：

- 每集有戏剧任务。
- 每集有本集阻力、本集误判/错位、本集只确认的信息、本集必须隐藏的信息、本集情绪或悬念推进方式。
- 每集有结尾钩子。
- 前 15 秒有抓人手段；冷开场可选但不能慢热。
- 分集依据来自断点、情绪转向、悬念节点或材料量，不按字数机械均分。
- 信息差或线索投放没有提前泄露。
- 相邻集结尾情绪不连续同质。

不合格返回：[outline-writer.md](outline-writer.md)。

## 4. 类型强度质量门

悬疑通过条件：

- 后半段每集有阻力、误判或人物选择代价。
- 证据不太完整，关键证据优先残缺、半真、可误读或延迟回收。
- 对峙不一次性解释核心真相。
- 工具人不整段说明真相、动机或线索含义。
- 第 6 集后不得连续靠文件、手机、邮件推进。
- 终局收人物选择和关系代价，不写成结案说明。

世情通过条件：

- 受压、崩溃、觉醒、反击都有具体触发。
- 冲突有具体压力，不只是争吵或情绪宣泄。
- 施压者有利益、面子、控制欲、亏欠或依赖逻辑。
- 觉醒/反击有前文信息、关系、资源或情绪临界点铺垫。
- 每 3-5 集有情绪峰值、关系反转或爽点兑现。
- 结尾回收尊严和代价，不只宣布胜利。

混合类型通过条件：

- 主类型完整通过对应强度门。
- 副类型至少通过补充检查，且不得破坏主类型节奏。

不合格返回：[outline-writer.md](outline-writer.md)，必要时回到 [worldly-drama-analyst.md](worldly-drama-analyst.md) 或 [suspense-analyst.md](suspense-analyst.md) 补强分析。

## 5. 单集剧本质量门

通过条件：

- 使用中国影视剧本传统格式。
- 本集事件链验证成立。
- `△` 只写可观测内容，不写心理推断、文学比喻或动机解释。
- VO/OS/闪回/蒙太奇均有触发点，且声画咬合。
- 有名角色首次入画有人物字幕条。
- 短信/微信/截图/文字记录的处理符合剧情功能：能电话化则电话化，承担证据、误会、延迟阅读、传播功能则保留为屏幕文字或道具。
- 结尾钩子可信，不靠角色反复变卦制造假悬念。
- 对白不过度解释，不复述前情、逐条答疑、抽象下结论或让工具人说明真相。
- 悬疑对峙有证据道具、回避、半真话和未解释问题。
- 世情高压/反击有具体要求或损失、主角忍耐、具体爆发触发和反击代价。

不合格返回：[script-writer.md](script-writer.md)，必要时同步回到 [outline-writer.md](outline-writer.md) 调整本集任务或钩子。

## 6. 全剧一致性质量门

通过条件：

- 角色、时间线、结局、支线未被随意改动。
- 全剧信息差、线索、反转和因果链与分析表一致。
- 角色称谓、身份、人物字幕条、地点和关键道具前后一致。
- 辅助文档与剧本内容一致。
- 输出目录符合默认 `outputs/{故事名}/` 或用户指定路径。

不合格返回：

- 角色/地点/道具不一致：回到 [script-writer.md](script-writer.md) 修订相关集和辅助文档。
- 信息投放错位：回到对应分析 agent 和 [outline-writer.md](outline-writer.md)。
- 结局、时间线、支线被改：回到对应分析 agent 重做事件清单或时间线。
