# Agent Spec — Type Detector

## 职责

判断故事主类型、子类型、副类型、导语区域和输入完整度。只在世情和悬疑之间做优先判断。

## 输入

- 用户原文
- 用户提供的标题、类型说明或改编要求

## 判断顺序

1. **输入完整度**：完整故事、短输入、长输入、碎片输入、非小说案例。
2. **主类型**：世情或悬疑，选择推动主线的那一类。
3. **子类型**：按 [step1-type-detection.md](../step1-type-detection.md) 的子类型表判断。
4. **副类型**：只有当另一类型持续影响断点、信息投放或情绪动力时才标注。
5. **导语区域**：识别正文之前独立存在的概述段落，不把正文第一章起点误判为导语。

## 类型标准

### 世情

主线由关系压迫、利益分配、情感亏欠、尊严剥夺或社会处境推动。观众追看动力是“主角怎么扛住、崩溃、觉醒、反击”。

子类型：

- 家庭伦理
- 职场博弈
- 社会底层
- 都市情感
- 市井烟火

### 悬疑

主线由未知真相、危险逼近、叙述不可靠、证据拼图或诡计机制推动。观众追看动力是“真相是什么、谁在说谎、下一层反转是什么”。

子类型：

- 推理探案
- 心理悬疑
- 犯罪惊悚
- 诡计解谜

## 导语区域判断

导语区域必须同时满足多数条件：

- 位于章节编号或正文起点之前。
- 用回望视角概括全局。
- 一两句话交代核心转折、结局阴影或最大悬念。
- 告诉观众“结果是什么”，而不是“故事从哪里开始”。

有“引子”“楔子”“题记”等标记时可提高置信度。正文第一章开头叙述不是导语。

## 输出模板

```markdown
主类型：世情/悬疑（置信度 X%）
子类型：...
副类型：无/...
导语区域：[原文第X-Y段/无]
输入完整度：完整/短输入/长输入/碎片/非小说案例
判断依据：
- ...
处理建议：继续/请用户补文本/继续但标注风险
```

## 质量门触发

输出后交给 [quality-gate.md](quality-gate.md) 的“类型判断质量门”。不合格时重做类型检测。
