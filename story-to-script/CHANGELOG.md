# 版本变更总结

## 2026-05-30 — workflow + agents spec 重构

### 变更目标

将 `story-to-script` 从单体 `SKILL.md` 改为“文档型 workflow + agents spec”结构，优先提升世情和悬疑两类短剧改编的稳定性、可执行性和可验证性。

### 核心保留原则

- 原文保真：角色、时间线、结局、支线不随意改。
- 文学叙述必须转为视听语言。
- 信息差保护：观众和角色的知情顺序不能被无意改变。
- 因果链保护：情绪爆发、对白、闪回、VO、反转都必须有前置触发。
- 爽点只能强化原文已有节点，不能凭空创造。

### 结构变化

- `SKILL.md` 改为入口和调度说明，不再承载全部细节。
- 新增 `workflow.md`，定义完整执行闭环：
  - 输入接收
  - 文本完整度判断
  - 类型检测
  - 世情/悬疑分支分析
  - 分集大纲
  - 试写一集校准
  - 批量生成
  - 质量检查
  - 文件落盘
- 新增 `agents/`：
  - `type-detector.md`
  - `worldly-drama-analyst.md`
  - `suspense-analyst.md`
  - `outline-writer.md`
  - `script-writer.md`
  - `quality-gate.md`
- 新增 `assets/` 模板：
  - `characters.md`
  - `appearance.md`
  - `locations.md`
  - `overview.md`
- 保留 `step1-type-detection.md`、`step2-story-analysis.md`、`step3-outline.md`、`step4-script-writing.md` 作为附录细则，供对应 agent 引用。

### 关键规则调整

- 输出路径不再写死到桌面 `scripts/`。
- 默认输出路径统一为当前工作区 `outputs/{故事名}/`。
- 用户明确指定路径时，才使用用户路径。
- “短信一律替换电话”改为剧情功能判断：
  - 默认优先电话或视频通话，增强声音、台词和情绪。
  - 短信、微信、截图、聊天记录、新闻标题、监控文字等如果承担证据、误会、延迟阅读、传播功能，必须保留为屏幕文字或道具。

### 质量门增强

新增 `quality-gate.md`，覆盖五类验收：

- 类型判断质量门
- 分析质量门
- 大纲质量门
- 单集剧本质量门
- 全剧一致性质量门

每个质量门都定义了通过条件和不合格时返回的重做步骤，避免流程继续带错。

### 世情分支增强

`worldly-drama-analyst.md` 明确要求输出：

- 人物关系压迫图
- 原文事件清单
- 情绪升级曲线
- 主角受压 / 崩溃 / 觉醒 / 反击节拍
- 信息差装置

目标是避免只按剧情流水拆集，而忽略世情短剧真正依赖的关系压迫和情绪转向。

### 悬疑分支增强

`suspense-analyst.md` 明确要求输出：

- 真相时间线
- 表面时间线
- 线索台账
- 误导层级
- 观众/角色知情矩阵
- 反转节奏表

目标是避免为了短剧节奏提前泄露线索、破坏误导层，或让反转缺少前置依据。

### 已完成验证

- 文件结构完整。
- Markdown 相对链接有效。
- `assets` 引用均有对应文件。
- 输出路径说明一致，已统一为 `outputs/{故事名}/` 或用户指定路径。
- 未发现旧的桌面 `scripts/` 输出路径残留。
- 未发现旧的“短信一律替换电话”强规则残留。
- `workflow.md` 能从输入接收到文件落盘形成闭环。
- `quality-gate.md` 覆盖不合格返回路径。

### 尚未验证

- 尚未使用 5000-12000 字世情或悬疑文章做完整内容试跑。
- 尚未验证真实调用时是否会稳定执行“试写一集校准”再批量生成。
- 尚未验证长文、多支线、混合类型故事下的质量门返工效果。

### 建议下一步

用新 Codex 窗口进行真实调用测试，输入一篇完整世情或悬疑文章，重点观察：

- 是否先做类型检测和输入完整度判断。
- 是否正确进入世情或悬疑分支分析。
- 是否按 3-5 集分段展示大纲。
- 是否先试写一集校准，再批量生成。
- 是否默认输出到 `outputs/{故事名}/`。
- 是否正确处理短信、微信、截图和文字证据。
